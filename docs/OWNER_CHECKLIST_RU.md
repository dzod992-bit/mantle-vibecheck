# Что требуется от владельца проекта

Ниже перечислены только действия, которые нельзя безопасно выполнить вместо
вас: создание аккаунтов, владение кошельком и публикация от вашего имени.

## 1. GitHub - нужно сейчас

1. Откройте `https://github.com` и войдите в свой аккаунт.
2. Справа сверху нажмите `+`, затем `New repository`.
3. В поле `Repository name` напишите `mantle-vibecheck`.
4. Выберите `Public`.
5. Не включайте `Add a README`, `.gitignore` и лицензию: эти файлы уже есть.
6. Нажмите `Create repository`.
7. Скопируйте HTTPS-адрес вида
   `https://github.com/ВАШЕ_ИМЯ/mantle-vibecheck.git`.
8. Передайте только этот публичный адрес. Пароль или токен передавать не
   нужно.

Также нужны имя и публичный email для Git commit. Email можно взять на GitHub:
`Settings` -> `Emails` -> адрес вида `12345+name@users.noreply.github.com`.

Готово: публичный репозиторий создан и подключён:
`https://github.com/dzod992-bit/mantle-vibecheck`.

## 2. DoraHacks - зарегистрироваться до финальной подачи

1. Откройте страницу хакатона:
   `https://dorahacks.io/hackathon/mantleturingtesthackathon2026`.
2. Нажмите `Register as Hacker`.
3. Войдите или создайте аккаунт.
4. Заполните публичный профиль реальными данными, которые готовы показывать
   судьям.
5. Пока не нажимайте финальную отправку BUIDL: ссылки и адрес контракта ещё
   будут меняться.

## 3. X / Twitter - нужен для submission и community voting

1. Нужен публичный аккаунт на `https://x.com`.
2. Добавьте понятное имя, фотографию или логотип и короткое описание.
3. Не публикуйте черновой проект до появления публичного demo.
4. В финале потребуется thread с тегом `#MantleAIHackathon`, pitch, видео,
   GitHub и адресом Mantle-контракта.

Публикацию делает владелец аккаунта. Мы подготовим готовый текст и порядок
изображений.

## 4. Кошелёк - нужен отдельный тестовый

1. Используйте MetaMask или Rabby.
2. Создайте отдельный кошелёк только для хакатона.
3. Сохраните seed phrase офлайн.
4. Никому, включая разработчика, не отправляйте seed phrase или private key.
5. Для разработки потребуется публичный адрес кошелька и немного testnet MNT.

Private key позже добавляется владельцем только в локальный `.env` или в
защищённые настройки deployment-сервиса. Он не должен попадать в чат, GitHub
или видео.

## 5. AI API

API key совместимого LLM-провайдера необязателен для запуска: без него работает
прозрачный deterministic fallback. Для полноценного live AI demo ключ
желателен.

Ключ нужно будет самостоятельно записать в `.env.local`:

```text
AI_API_KEY=...
AI_MODEL=...
AI_API_BASE_URL=https://api.openai.com/v1
```

Файл уже исключён из Git.

## 6. Публичный deployment

Для frontend рекомендуется Vercel:

1. Откройте `https://vercel.com`.
2. Войдите через тот же GitHub.
3. На финальной итерации нажмите `Add New` -> `Project`.
4. Выберите репозиторий `mantle-vibecheck`.
5. Секреты добавляются в `Settings` -> `Environment Variables`.

До подключения GitHub ничего создавать на Vercel не требуется.

## Что нужно передать в первую очередь

- Создать бесплатный Etherscan API key для автоматической verification:
  `https://etherscan.io/myapikey`.
- Записать demo video и опубликовать X thread после появления public URL.

Не передавайте пароли, seed phrase, private key, recovery codes или API keys в
открытом виде.
